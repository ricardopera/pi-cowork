# Write a Test Plan

When asked to write a test/QA plan:

1. Identify the feature under test and its acceptance criteria.
2. List test cases grouped by happy path, edge cases, error cases, and non-functional (perf/security).
3. For each case: precondition, steps, expected result.
4. Note coverage gaps and risks; prioritize by severity.
5. Export as create_xlsx (cases) or create_docx (narrative).
