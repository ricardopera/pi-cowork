# Write a Design Doc

When asked to write a design doc/RFC:

1. Capture Context (why now), Goals/Non-goals, and a one-paragraph summary.
2. Describe the proposed design with a diagram (mermaid) and key components.
3. List alternatives considered and why they were rejected.
4. Cover security, performance, migration, and rollout.
5. Call out open questions explicitly.
6. Export as create_markdown or create_docx.
