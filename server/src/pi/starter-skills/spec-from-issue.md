# Spec from Issue

When asked to turn an issue into a spec:

1. Restate the problem in one sentence; quote the reporter's goal.
2. Define acceptance criteria as testable bullets.
3. Note scope boundaries (what's explicitly out of scope).
4. List unknowns/dependencies that need answers.
5. Export as create_markdown ready to paste into the ticket.
