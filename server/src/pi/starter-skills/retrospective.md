# Retrospective

When asked to run/summarize a retro:

1. Gather input (survey results, notes, metrics).
2. Organize into: What went well → What didn't → What to try next.
3. Convert each "didn't" into a concrete action with an owner.
4. Keep it blameless — focus on systems/processes, not people.
5. Export as create_docx; save durable action items to memory.
