# Write a README

When asked to write a README:

1. Capture: what it is, why use it, how to install, how to use, how to develop.
2. Lead with a one-line summary + a quick-start snippet.
3. Include a features list, configuration, and common pitfalls.
4. Add license + contributing notes.
5. Export as create_markdown; keep it skimmable.
