# Review a Pull Request

When asked to review a PR/diff:

1. Read the diff (bash `git diff` or the provided patch).
2. Understand the intent (read the description/linked issue).
3. Comment on: correctness, edge cases, security, tests, readability.
4. Distinguish must-fix from nits; lead with what's good.
5. Summarize: approve / request changes / needs discussion.
