# Write an RFP Response

When asked to respond to an RFP/questionnaire:

1. Read the requirements; map each to evidence (features, docs, certs).
2. Answer each requirement: Comply / Partially / Not supported, with specifics.
3. Don't overclaim — flag genuine gaps and offer workarounds.
4. Keep language consistent; reuse approved boilerplate where possible.
5. Export as create_xlsx (requirements matrix) + create_docx (narrative).
