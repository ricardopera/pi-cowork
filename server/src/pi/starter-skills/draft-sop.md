# Write an SOP

When asked to write an SOP:

1. State the purpose, scope, and who owns it.
2. List prerequisites and required access/tools.
3. Number each step with exact actions and expected results.
4. Include troubleshooting for known failure modes.
5. Add a revision date + owner; export as create_docx.
