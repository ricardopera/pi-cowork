# Write a Changelog

When asked to write/maintain a changelog:

1. Follow Keep a Changelog: Added / Changed / Deprecated / Removed / Fixed / Security.
2. One section per released version, newest first, with dates.
3. Link each entry to its issue/PR where possible.
4. Distinguish breaking changes with a **BREAKING** marker.
5. Export as create_markdown (CHANGELOG.md).
